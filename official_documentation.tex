\section{Executive Summary}

\vspace{0.3cm}\hrule\vspace{0.6cm}

The HPE Ring Test Management platform is a full-stack enterprise SAN discovery and infrastructure intelligence system. It combines virtual SAN simulation, breadth-first topology discovery, Neo4j graph persistence, Elasticsearch indexing, and AI-assisted operational workflows in a single monorepo.

Core capabilities include:

\begin{itemize}[leftmargin=1.5cm]
    \item Virtual HPE array, switch, and host emulation without physical hardware
    \item Live discovery with streaming events into graph and search indexes
    \item React operational dashboards for topology, inventory, health, and administration
    \item Node.js chatbot service with SAN agent, GraphRAG, and dual LLM engines (Groq/Gemini)
    \item Flask REST orchestration across simulator, discovery, graph, and AI proxy layers
    \item Docker Compose for Neo4j, Elasticsearch, and MongoDB
\end{itemize}

\section{Platform Overview}

\vspace{0.3cm}\hrule\vspace{0.6cm}

The monorepo unifies \texttt{simulator/}, \texttt{discovery/}, \texttt{api/}, \texttt{chatbot-service/}, and \texttt{dashboard/} into one deployable environment. Operators interact primarily through the React dashboard (\texttt{http://localhost:5173}); all business logic routes through the Flask API on port \textbf{5005}, with the chatbot on \textbf{5010} and the simulator on \textbf{5001}.

\subsection{End-to-End Data Flow}

\begin{enumerate}[leftmargin=1.5cm]
    \item The \textbf{simulator} exposes virtual devices and CLI responses over REST.
    \item \textbf{Discovery} crawls the virtual network (BFS), fingerprints devices, parses metadata, and streams progress.
    \item The \textbf{Flask API} persists relationships to \textbf{Neo4j} and documents to \textbf{Elasticsearch}.
    \item The \textbf{dashboard} visualizes graphs, runs terminals, and surfaces health and admin tools.
    \item The \textbf{chatbot} retrieves graph and index context for SAN-aware answers and agent plans.
\end{enumerate}

\section{Dashboard User Guide}

\vspace{0.3cm}\hrule\vspace{0.6cm}

The React dashboard (\texttt{dashboard/src/}) is the primary operator interface. It authenticates against the chatbot service, proxies API calls to Flask, and exposes seven functional areas: \textbf{Discovery}, \textbf{Test Ring Viewer}, \textbf{Inventory}, \textbf{Emulator}, \textbf{AI Assistant}, \textbf{Admin} (admin role only), and \textbf{Health}.

Navigation is defined in \texttt{App.jsx}; the status bar polls \texttt{/api/health} to show Flask, simulator, and chatbot availability.

% ----- AUTH -----
\subsection{Authentication and Sign-Up}

\textbf{Frontend.} \texttt{LoginPage.jsx} provides login and registration with role selection (team member, team lead, senior manager, admin), team/cluster assignment, and password validation.

\textbf{Backend.} The chatbot service (\texttt{chatbot-service/}) exposes \texttt{POST /api/auth/login} and \texttt{POST /api/auth/register}; credentials and roles are stored in MongoDB. JWT tokens gate protected dashboard routes.

\screenshot{Sign-up.png}{Sign-up screen: new user registration with role, team, and cluster assignment before entering the operational dashboard.}{signup}

% ----- DISCOVERY -----
\subsection{Discovery Tab}

\textbf{Route:} \texttt{/discovery} \quad \textbf{Component:} \texttt{DiscoveryPage.jsx}

The Discovery tab is the default landing view after login. It orchestrates live infrastructure scanning and visualization.

\subsubsection{Frontend Features}

\begin{itemize}[leftmargin=1.5cm]
    \item \textbf{Discovery panel} --- seed IP entry, start/stop controls, and SSE-powered live event log (\texttt{DiscoveryPanel.jsx})
    \item \textbf{Topology canvas} --- interactive graph with node selection, pulsing highlights for new devices (\texttt{TopologyCanvas.jsx})
    \item \textbf{Neural graph view} --- alternate visualization mode for scan progress (\texttt{NeuralGraph.jsx})
    \item \textbf{Result grid} --- tabular list of discovered entities (\texttt{DiscoveryResultGrid.jsx})
    \item \textbf{Aggregate sidebar} --- counts by device type (\texttt{AggregateSidebar.jsx})
    \item \textbf{Embedded terminal} --- open CLI on any discovered node (\texttt{NodeTerminal.jsx})
    \item \textbf{Session persistence} --- graph and events restored via \texttt{sessionStorage} across refreshes
\end{itemize}

\subsubsection{Backend Integration}

\begin{itemize}[leftmargin=1.5cm]
    \item \texttt{POST /api/discovery/start} --- initiates BFS crawl from seed IP (default \texttt{10.20.10.5})
    \item \texttt{GET /api/discovery/stream} --- Server-Sent Events: connecting, command output, parsed entities, \texttt{neo4j\_stored}, \texttt{es\_indexed}
    \item \texttt{GET /api/graph/neo4j} --- loads persisted graph for canvas rendering
    \item Simulator on port 5001 --- emulates device CLI responses during crawl
\end{itemize}

\screenshot{discovery-tab-fullscreen.png}{Discovery tab (fullscreen): live BFS network scan with interactive topology graph, discovery controls, and real-time event stream.}{discovery-tab}

% ----- TOPOLOGY -----
\subsection{Test Ring Viewer Tab}

\textbf{Route:} \texttt{/topology} \quad \textbf{Component:} \texttt{TopologyPage.jsx}

The Test Ring Viewer presents the SAN test ring from both diagrammatic and graph perspectives, with role-based visibility (admin, manager, user teams from \texttt{teamconfig.json}).

\subsubsection{Frontend Features}

\begin{itemize}[leftmargin=1.5cm]
    \item \textbf{SAN diagram} --- hierarchical layout of arrays, FC switches, Ethernet switches, and hosts (\texttt{SANDiagram.jsx})
    \item \textbf{Graph canvas} --- React Flow topology with focus, expand, and multi-select (\texttt{TopologyCanvas.jsx})
    \item \textbf{Node card} --- firmware, ports, team/cluster metadata on selection (\texttt{NodeCard.jsx})
    \item \textbf{Decommission workflow} --- mark devices offline and manage ring membership
    \item \textbf{Topology import/export} --- JSON import and download for offline analysis
    \item \textbf{Role/team filters} --- managers see assigned teams; admins see all clusters
\end{itemize}

\subsubsection{Backend Integration}

\begin{itemize}[leftmargin=1.5cm]
    \item \texttt{GET /api/ontology/topology} and \texttt{GET /api/graph/neo4j} --- primary data sources
    \item Fallback to remote ontology service if local graph is empty
    \item Graph mutations via admin/decommission APIs where applicable
\end{itemize}

\subsection{Emulator Tab}

\textbf{Route:} \texttt{/emulator} \quad \textbf{Component:} \texttt{EmulatorPage.jsx}

The Emulator (labeled ``Emulator'' in navigation) is the device terminal workspace for executing CLI commands against simulated infrastructure without running a full discovery scan.

\subsubsection{Frontend Features}

\begin{itemize}[leftmargin=1.5cm]

\subsubsection{Backend Integration}

\begin{itemize}[leftmargin=1.5cm]
    \item \texttt{POST /api/simulator/exec} --- executes commands on virtual devices (port 5001)

\textbf{Route:} \texttt{/chat} \quad \textbf{Component:} \texttt{ChatPage.jsx}

The AI Assistant is a full-page conversational interface backed by the Node.js chatbot and Flask RAG proxy. It supports operational questioning across the SAN test ring.

\subsubsection{Frontend Features}

\begin{itemize}[leftmargin=1.5cm]
    \item \textbf{SAN agent mode} --- multi-step plans with \texttt{AgentStepTimeline} and \texttt{AgentReasoningSidebar}

\begin{itemize}[leftmargin=1.5cm]
    \item Chatbot \texttt{/api/chat} (and streaming variants) --- orchestrates LLM calls
    \item Flask \texttt{/api/chat} --- hybrid RAG: Elasticsearch context + Neo4j graph traversal
    \item Neo4j --- graph-aware answers (zones, peers, array relationships)
    \item Elasticsearch --- indexed device and discovery metadata for retrieval

\section{Backend Services}

\vspace{0.3cm}\hrule\vspace{0.6cm}

\subsection{Simulator Subsystem}

Python modules under \texttt{simulator/} emulate HPE arrays, FC switches, Linux/Windows hosts, and virtual IP routing. \texttt{simulator\_manager.py} boots the fleet; \texttt{device\_terminal.py} and \texttt{proxy\_engine.py} handle CLI sessions. REST on port \textbf{5001} serves the discovery crawler and dashboard emulator.

\subsection{Discovery Engine}

The \texttt{discovery/} package runs BFS crawling, multi-vendor fingerprinting, command execution, parsing, Neo4j persistence, and Elasticsearch indexing. Events streamed to the dashboard match the color-coded types in \texttt{DiscoveryPanel.jsx} (\texttt{neo4j\_stored}, \texttt{es\_indexed}, etc.).

\subsection{Flask REST API}

\texttt{api/app.py} is the integration hub. Representative endpoints:

\rowcolors{2}{tablegray}{white}
\begin{longtable}{p{5.2cm} p{8.8cm}}
\toprule
\rowcolor{headerbg}
\textbf{Endpoint} & \textbf{Purpose} \\
\midrule
\texttt{/api/discovery/start} & Start BFS discovery \\
\texttt{/api/discovery/stream} & SSE discovery events \\
\texttt{/api/simulator/exec} & Remote CLI on virtual device \\
\texttt{/api/graph/neo4j} & Topology graph JSON \\

\texttt{/api/search} & Elasticsearch queries \\

\subsection{AI and RAG Pipeline}

\begin{enumerate}[leftmargin=1.5cm]
    \item User query enters chatbot or Flask \texttt{/api/chat}
    \item Query classification selects agent, standard, or GraphRAG path
    \item Elasticsearch returns relevant device/index chunks
    \item Neo4j supplies relationship context (peers, zones, containment)
    \item Groq or Gemini synthesizes the answer; agent mode may loop with simulator exec
    \item Response streams to \texttt{ChatPage.jsx} with optional step timeline
\end{enumerate}

\section{Graph and Search Persistence}

\vspace{0.3cm}\hrule\vspace{0.6cm}

\subsection{Neo4j}

Node labels include \texttt{ArraySystem}, \texttt{Node}, \texttt{Host}, \texttt{Switch}, \texttt{Cage}, and \texttt{PhysicalDisk}. Relationships include \texttt{HAS\_NODE}, \texttt{HAS\_CAGE}, \texttt{CONNECTS\_TO}, \texttt{CONTAINS}, \texttt{REMOTE\_COPY\_PEER}, and \texttt{PEER}. The AI Assistant and Test Ring Viewer depend on this graph for accurate SAN reasoning.

\subsection{Elasticsearch}

Discovery indexes device identifiers, firmware, capacity, topology summaries, and timestamps. RAG queries use this index before graph enrichment.

\subsection{MongoDB}

Stores authentication records, chat sessions, and operational metrics for the chatbot service.

\section{Troubleshooting}

\vspace{0.3cm}\hrule\vspace{0.6cm}

Use the \textbf{Health} tab (Figure~\ref{fig:health-services}) to identify which dependency is down. Common fixes:

\begin{itemize}[leftmargin=1.5cm]
    \item \textbf{Red Flask API} --- ensure \texttt{py api/app.py} is running and port 5005 is free; stop Docker \texttt{hpe\_san\_api} if present
    \item \textbf{Red Simulator} --- run \texttt{py simulator/simulator\_manager.py} or \texttt{npm start}
    \item \textbf{Red Neo4j/ES} --- \texttt{docker-compose up -d neo4j elasticsearch}; check \texttt{docker logs}
    \item \textbf{Discovery seed fails} --- simulator must be on 5001; default seed \texttt{10.20.10.5}
    \item \textbf{Chatbot errors} --- verify \texttt{GROQ\_API\_KEY} in root \texttt{.env}
\end{itemize}
